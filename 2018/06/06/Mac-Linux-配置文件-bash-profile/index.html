Mac & Linux 配置文件 bash_profile

Posted on 2018-06-06 | In linux | Visitors: | 浏览 次
Words count in article: 662 字 | Reading time ≈ 2 分钟

最近开始找实习了，很多事情很忙，所以博客很久没更了。在笔试的过程中发现要考 Python，很久以前学过，但是感觉自己基本忘光。所以从头捡起来吧。给大家一个建议，在学生阶段最好要有一条主线任务。像我就是支线剧情打太多，技能🌲都点了，没有太深入的，所以感觉现在找实习的时候就比较黑了。好啦啰嗦的话说完了，下面主要介绍一下 Mac 的配置文件加载，环境变量啊，什么的。Linux 和 Mac 类似，大家可以扩展阅读一下，这里就不展开细说了。

我认为程序员用 Mac 做开发要比用 Windows 本更好一点，因为自带了很多插件，比如 Mac 就安装了 Python，版本是稳定的2.7。但是因为是系统自带的，所以也不允许用户自己修改。我这么爱折腾的人，管他有用没用，我都想改改，有点自己的影子才好。所以安装了新的3.5的版本，但是悲剧的是不管我怎么调，也没有办法改变当前 Python 的编译环境。最后在知乎上有大神给意见，在全局变量里面修改。因此下面介绍一下 Mac 的配置文件加载的问题。

Mac系统的环境变量，加载顺序为：

当然/etc/profile和/etc/paths是系统级别的，系统启动就会加载，后面几个是当前用户级的环境变量。后面3个按照从前往后的顺序读取，如果~/.bash_profile文件存在，则后面的几个文件就会被忽略不读了，如果~/.bash_profile文件不存在，才会以此类推读取后面的文件。~/.bashrc没有上述规则，它是bash shell打开的时候载入的。

全局设置

下面的几个文件设置是全局的，修改时需要root权限

1）/etc/paths （全局建议修改这个文件 ）

编辑 paths，将环境变量添加到 paths文件中 ，一行一个路径

Hint：输入环境变量时，不用一个一个地输入，只要拖动文件夹到 Terminal 里就可以了。

2）/etc/profile （建议不修改这个文件 ）

全局（公有）配置，不管是哪个用户，登录时都会读取该文件。

3）/etc/bashrc （一般在这个文件中添加系统级环境变量）

全局（公有）配置，bash shell执行时，不管是何种方式，都会读取此文件。
